# ARG-Expert: Integrating Pre-trained Protein Language Models with Transformer Architectures for Antibiotic Resistance Gene Identification and Classification

## Overview

ARG-Expert is a deep learning framework for accurate identification and classification of Antibiotic Resistance Genes (ARGs). It combines a pre-trained protein language model (ESM2) with a custom Transformer through multi-task learning to simultaneously perform:

1. **Binary Classification**: ARG vs. non-ARG
2. **Multi-class Classification**: 14 resistance categories (beta-lactam, multidrug, MLS, aminoglycoside, etc.)

## Architecture

```
Input Sequence
    |
Multi-Scale Embedding (ESM2-150M + Learnable)
    |
Transformer Encoder (4 layers, 8 heads, relative positional encoding)
    |
Gated Residual Fusion
    |
Multi-Task Heads (decoupled binary + 14-class)
```

- Python >= 3.10
- CUDA-capable GPU (recommended; V100S 32 GB used in development)
- Conda (recommended for environment management)

```bash
# Create and activate conda environment
conda create -n gene_pred python=3.10 -y
conda activate gene_pred

# Install dependencies
pip install -r docs/requirements.txt
```

## Quick Start: Inference

Use our pre-trained model or your own checkpoint to predict ARGs from protein sequences.

### Command Line

```bash
# FASTA input
python model/inference.py \
    --model output/best_model_warmup.pt \
    --input sequences.fasta \
    --input_type fasta \
    --output predictions.json

# CSV input
python model/inference.py \
    --model output/best_model_warmup.pt \
    --input sequences.csv \
    --input_type csv \
    --sequence_column sequence \
    --output predictions.json
```

Or use the shell script:

```bash
chmod +x scripts/predict.sh
./scripts/predict.sh -i sequences.fasta -o predictions.json
```

## Training

### 1. Prepare Data

The ARG positive sequences are provided in `data/ARG_DB.fasta`. You need to supply your own negative (non-ARG) protein sequences before preprocessing. Place them as:

- `data/Non_ARG_DB.fasta` — smaller negative set (e.g., ~180K non-ARG sequences)
- `data/GCF_negatives.fasta` — larger metagenomic negative set (optional)

Or provide custom paths:

```bash
conda activate gene_pred
python scripts/prepare_data.py \
    --neg_fasta_a data/Non_ARG_DB.fasta \
    --neg_fasta_b data/GCF_negatives.fasta
```

The preprocessing pipeline performs:
- Category mapping: 30 raw categories → 14 classes
- CD-HIT 70% clustering for homology-based train/val/test split
- MMseqs2 hard-negative mining against training ARGs

After preprocessing, the generated data will be in `data/processed/`.

### 2. Train the Model

```bash
chmod +x scripts/train_optimized.sh
./scripts/train_optimized.sh
```

### 3. Evaluate

```bash
python model/evaluate_entry.py \
    --model output/best_model_warmup.pt \
    --data_path data/processed \
    --output_dir output
```

## Data

### Source Sequences

| File | Description |
|------|-------------|
| `data/ARG_DB.fasta` | 17,345 ARG positive sequences (30 raw categories, merged into 14 classes) |

Negative sequences are **not included** — you must supply your own non-ARG protein database. The preprocessing script accepts up to two negative FASTA sources:

| Placeholder Path | Expected Content |
|------------------|------------------|
| `data/Non_ARG_DB.fasta` | Curated non-ARG protein sequences |
| `data/GCF_negatives.fasta` | Large-scale metagenomic non-ARG sequences (optional) |

Alternatively, provide custom paths via `--neg_fasta_a` and `--neg_fasta_b`.

### Processed Data (generated by `prepare_data.py`)

After running the preprocessing pipeline, `data/processed/` will contain:

| File | Description |
|------|-------------|
| `train.csv` | Training set (~13,870 ARG + ~41,610 negatives) |
| `val.csv` | Validation set (~1,729 ARG + ~5,187 negatives) |
| `test.csv` | Test set (~1,746 ARG + ~5,238 negatives) |
| `train_neg_pool.csv` | Hard negative pool from MMseqs2 screening |
| `preprocessing_report.json` | Category distribution and split statistics |

All splits are cluster-level (CD-HIT 70%) to prevent homologous data leakage.
